🔁 Student Application Workflow Statuses
1. Draft / Incomplete
The student has started the application but hasn’t submitted it.

Action Required: Applicant needs to complete and submit.

2. Submitted
The application has been submitted successfully.

Action Required: Awaiting processing by admissions staff.

3. Under Review / In Review
The admissions team is currently reviewing the submitted documents.

Action Required: None (for applicant), processing by staff.

4. Awaiting Documents
Some required documents are missing or incomplete.

Action Required: Applicant must upload/provide additional documents.

5. Awaiting Payment
The application fee hasn’t been paid (if applicable).

Action Required: Applicant must complete payment.

6. Interview Scheduled / Pending Interview
An interview has been scheduled or is required.

Action Required: Applicant must attend the interview.

7. Interview Completed
Interview conducted successfully.

Action Required: None; awaiting decision.

8. Decision Pending
All steps completed, waiting for final decision.

Action Required: Internal review/approval by admissions board.

9. Accepted / Offer Made
The student has been accepted.

Optional Sub-status:

Conditional Acceptance (pending exam results, documents, etc.)

Unconditional Acceptance (all requirements met)

10. Waitlisted
The applicant meets requirements but there’s no space currently.

Action Required: Wait for updates from the admissions team.

11. Rejected
The application has been declined.

Action Required: None; optionally notify applicant with reason.

12. Offer Accepted
Applicant has accepted the admission offer.

Action Required: Proceed to enrollment/registration.

13. Offer Declined
Applicant declined the admission offer.

Action Required: Close the application.

14. Enrolled / Registered
The student has officially enrolled in the program.

End of workflow (often transitions into Student Information System).
